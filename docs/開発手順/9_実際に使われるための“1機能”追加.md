# 開発手順 9: 実際に使われるための「1機能」追加

MVPが動いたあとに「使われる確率」を上げる、追加しやすい1機能案を整理します。

## 候補（実装コスト低→効果高）

- お気に入り（スター）
  - クエリ/ダッシュボードに `is_favorite` を追加

- 共有リンク（読み取り専用）
  - ダッシュボードをトークン付きで公開（MVPの外でも可）

- 直近実行の履歴表示
  - `query_executions` をUIで見せる

## まずはこれ（推奨）

- 「直近実行履歴」
  - 既に `query_executions` があるため、UI追加だけで価値が出る
