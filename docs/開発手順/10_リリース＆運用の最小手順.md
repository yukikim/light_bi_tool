# 開発手順 10: リリース＆運用の最小手順

デモ〜小規模運用を想定した最小手順です。

## 1. 依存

- Frontend: Next.js（Vercel等）
- Backend: NestJS（Fly.io / Railway等）
- DB: Managed PostgreSQL

## 2. 環境変数（最低限）

Backend:
- `DATABASE_URL`
- `JWT_SECRET`
- `EXECUTE_TIMEOUT_MS`
- `MAX_RESULT_ROWS`

Frontend:
- `BACKEND_API_BASE_URL`（Next API Routes がバックエンドにプロキシするため）

## 3. 運用でやること（最低限）

- statement_timeout / max rows の値を保守的に
- エラーログ（500/BadRequest多発）を監視
- DBバックアップ（Managed側の自動バックアップを利用）

## 4. リリース前チェック

- SQL安全チェックが想定通り効く
- CSVアップロードで異常系（空、列数不一致、行数多すぎ）を弾ける
- クエリ削除の挙動（参照あり→確認→強制削除）
